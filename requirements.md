# F1レギュレーション変遷サイト 要件定義書

**バージョン：** v3.0（最終確定版）  
**作成日：** 2026年5月

---

## 1. プロジェクト概要

F1（フォーミュラ1）の2000年〜2026年のレギュレーション変遷を、F1ファン向けに公開する情報サイト。単年閲覧と2年度比較を核とした完成形に近いMVPをまず構築し、Vercel / Netlifyで一般公開する。

---

## 2. 想定ユーザー・公開方針

| 項目 | 内容 |
|------|------|
| 対象ユーザー | F1ファン一般 |
| 公開方法 | インターネット上に公開（Vercel または Netlify） |
| 開発方針 | まずMVPを作成し、その後段階的に拡張する |
| 対応デバイス | PCメイン（スマートフォン対応は将来拡張） |
| 言語 | 日本語のみ（多言語対応は将来拡張） |

---

## 3. 機能要件

### 3-1. 単年レギュレーション表示（画面A）

| 項目 | 仕様 |
|------|------|
| 年度選択 | ヘッダーのプルダウンで2000〜2026年を選択 |
| 車体画像 | 選択年に対応した画像を画面中央に表示 |
| レギュレーション | 8カテゴリをアコーディオン形式で表示 |
| アコーディオン初期状態 | 全カテゴリ折りたたみ済み |
| コンテンツ | マークダウンファイル（.md）を読み込んで表示 |

### 3-2. 2年度比較表示（画面B）

| 項目 | 仕様 |
|------|------|
| 年度選択 | 「Year A」「Year B」2つのプルダウンで選択 |
| 車体画像 | 比較する2年それぞれに対応した画像を上部に横並びで表示 |
| レイアウト | 車体画像の下に、カテゴリごとに左右2カラムで並べて表示 |
| 差異表示 | 各カテゴリ下部に「変更点サマリー（概要文）＋ 箇条書き詳細」を表示 |
| 差異データ | diffs.json（351通り）を参照 |
| 時系列表示 | 差異は常に「古い年 → 新しい年」の時系列順で表示（ユーザーが逆順で選択しても自動修正）|
| 時系列表示の注記 | 画面上に「※時系列順（XXXX年 → YYYY年）で表示しています」と明記 |

## 4. 車体画像マッピング

単年表示・比較表示の両画面で共通して使用する。

| 選択年 | 表示画像ファイル名 |
|--------|-----------------|
| 2000〜2008年 | `CarImage_2000-2008.png` |
| 2009〜2011年 | `CarImage_2009-2011.png` |
| 2012〜2021年 | `CarImage_2012-2021.png` |
| 2022〜2025年 | `CarImage_2022-2025.png` |
| 2026年 | `CarImage_2026.png` |

**例：** 2020年と2026年を比較する場合は `CarImage_2012-2021.png` と `CarImage_2026.png` を横並びで表示する。

```markdown
## ③ 空力・タイヤ・ホイール
（本文）

## ④ レースルール（Format）
（本文）

## ⑤ 安全・健康
（本文）

## ⑥ コスト・開発制限
（本文）

## ⑦ メディア・運営・公平性
（本文）

## ⑧ 環境・サステナビリティ
（本文）
```

### ファイル命名規則

- ファイル名は4桁の西暦のみ（例：`2000.md`、`2026.md`）
- 記号・スペース不要

## 10. diffs.jsonの構造

```json
{
  "2020_2026": {
    "engine": {
      "summary": "概要文（1〜2文）",
      "details": [
        "変更点1",
        "変更点2"
      ]
    },
    "chassis":     { "summary": "...", "details": [...] },
    "aero":        { "summary": "...", "details": [...] },
    "race_rules":  { "summary": "...", "details": [...] },
    "safety":      { "summary": "...", "details": [...] },
    "cost":        { "summary": "...", "details": [...] },
    "media":       { "summary": "...", "details": [...] },
    "environment": { "summary": "...", "details": [...] }
  }
}
```

**キー命名規則：** `"小さい年_大きい年"` の形式で統一（例：`"2000_2026"`）

## 11. 技術スタック

| 項目 | 採用技術 | 理由 |
|------|---------|------|
| フロントエンド | HTML / CSS / JavaScript | サーバー不要・管理が簡単 |
| マークダウン解析 | marked.js（CDN） | .mdを直接HTMLに変換 |
| ホスティング | Vercel または Netlify | 無料・GitHubと連携・自動デプロイ |
| バージョン管理 | GitHub | Vercel/Netlifyとの連携に必要 |

## 14. 注意事項・制約

| # | 内容 |
|---|------|
| 1 | 差異表示は常に「古い年 → 新しい年」の時系列順。ユーザーが逆順で選択した場合もシステムが自動で正しい順に修正し、画面上に時系列順であることを明記する |
| 2 | diffs.jsonのキーは必ず `小さい年_大きい年` の形式（例：`2000_2026`） |
| 3 | .mdファイルのカテゴリ見出し（`##`）の文言は全27ファイルで完全に統一する |
| 4 | diffs.jsonは `diffs/` フォルダ内に配置する |
| 5 | 画像ファイルは `images/` フォルダ内に配置する |
| 6 | レギュレーション原稿の内容はオーナーが最終確認を行う（特に数値情報） |
